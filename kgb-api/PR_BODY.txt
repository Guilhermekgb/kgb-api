feat: S3 presign uploads + local fallback for fotosClientes

Resumo
- Adiciona suporte opcional a uploads via presigned S3 URLs.
- Endpoint: `POST /fotos-clientes/presign` -> retorna `{ presignUrl, publicUrl }` quando as variáveis AWS estiverem configuradas.
- Fallback local: `POST /fotos-clientes/upload` grava em `public/uploads/<tenant>/...` e retorna URL local.
- Cliente: `public/api/storage-adapter.js` tenta presign -> PUT direto no S3; se falhar, faz upload ao endpoint local.
- Quando o frontend enviar `PATCH /fotos-clientes` com `value: null`, o servidor remove o arquivo local (se apontava para `public/uploads/...`) e dispara exclusão assíncrona no S3 se a URL pertencer ao bucket configurado.

Files changed (principais):
- `server.js` (novos endpoints `/fotos-clientes/presign` e lógica para remoção de arquivos em null)
- `public/api/storage-adapter.js` (fluxo presign + fallback upload)
- `package.json` (dependências AWS SDK adicionadas)
- `docs/cloud-poc.md` (documentação atualizada)
- `tests/delete-local-file-test.js` (novo teste que valida remoção local)

Notas técnicas
- A exclusão S3 é fire-and-forget e loga falhas no servidor; o mapa é atualizado para `null` mesmo se a deleção falhar.
- O fluxo é incremental e backward-compatible: o frontend continua funcionando com `localStorage` enquanto o adapter tenta sincronizar para o backend.
