Title: Draft PR — storage: fotosClientes → cloud + shim

Body:
This branch `feat/storage-cloudinary-enforce` contains the first non-destructive
steps to migrate `fotosClientes` from browser `localStorage` to a backend/cloud
flow (Cloudinary POC). Changes in this branch include:

- Server: Cloudinary-first upload endpoint and presign support (POC).
- Public client adapter: `public/api/storage-adapter.js` with `preload`, `patchFotos` and in-memory cache for `fotosClientes`.
- Shim: `public/js/fotos-shim.js` to provide a synchronous-safe getter and warm the in-memory cache.
- Report: `reports/fotosClients-scan.md` with remaining sites using sync localStorage reads.
- Minor fixes to avoid writing `fotosClientes` to `localStorage` and to patch client code.

Testing checklist (suggested):
- Start server locally and run `kgb-api/scripts/test-firebase-upload.ps1` to validate upload to Cloudinary.
- Open main pages (`cadastro-evento`, `evento-detalhado`, `area-cliente`, `lista-evento`) and verify photos still render without flash-of-empty-state.
- Confirm no sensitive base64 blobs are persisted in browser localStorage for `fotosClientes`.

Notes:
- Top-level HTML files were updated locally in the workspace to include the shim; if these files live in another repo, see `kgb-api/reports/html-includes-fotos-shim.patch` or run `kgb-api/scripts/apply-fotos-shim.ps1` to apply the changes.

Please review and if OK, merge to main after validating in staging.
